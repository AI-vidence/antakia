California housing tutorial

In this exemple, imagine you're are a real estate agent and you want to understand what drives the real estate prices in California.

The dataset you will use for this exemple is the California housing which is a dataset to test the regression model.

As you see there are 8 variables:

  • MedInc = Median income
  • HouseAge = House Age
  • AveRooms = Average rooms
  • AveBedrms = Average Bedrooms
  • Population = Population
  • AveOccup = Average Occupation
  • Latitude = Latitude
  • Longitude = Longitude

See our tutorial to go through this notebook.

On this interface there is:

  • (1) The values space
  • (2) The explanations space
  • Two scrolling list on the top:
      • (3) Projection in the VS
      • (4) Projection in the ES

Each of the projection list contain different method to view the point: * PCA * TSNE
UMAP
 PaCMAP

  • Three tab on the bottom:

      • (5) Selection
      • (6) Regions
      • (7) Substitution

  • (8) The button slider on the top left turn the two spaces from 2D -> 3D and 3D -> 2D:

1. Selection

To select an region of point you can use two tools: * The lasso selection * The box selection

Note: by default the lasso tool is selected

⚠ Double-click on the gray area to deselect.

⚠ Try to select an region where the zone coincide with the other spaces

2. Define Rules

Each of the different variables represent the main characteristics within the selected area.

⚠ You can adjust rules using the various sliders for a more precise selection.

After refining rules, click "Validate Rules" to confirm your selection criteria.

3. Region Operations

Click on the selected region than choose from the following options:

  • Delete
  • Substitute
  • Auto-Clustering

Delete remove the selected region.

Substitution send you to the Substitution tab

Auto-Clustering automatically selected every other region :

4. Substitution

Choose a substitution model (sub-model) from the available options. Each sub-model is accompanied by a percentage, indicating its functionality. Higher percentages imply better performance.
